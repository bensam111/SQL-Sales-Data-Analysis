# SQL-Sales-Data-Analysis
SQL Sales Data Analysis project with joins, subqueries, CTEs, window functions, and real business insights

📊 SQL Sales Data Analysis Project
📌 Overview

This project demonstrates how SQL can be used to analyse business data and generate meaningful insights. The analysis focuses on customer behaviour, product performance, and sales trends using a structured relational database.

🗂️ Dataset

The database consists of three main tables:

Customers – stores customer information (name, email, region)
Products – contains product details (name, category, price)
Sales – records transaction data (quantity, price, date)

These tables are linked using primary and foreign keys to ensure data integrity.

🛠️ SQL Skills Demonstrated

This project applies a wide range of SQL techniques:

Joins (INNER, LEFT, RIGHT, FULL OUTER)
Subqueries
Common Table Expressions (CTEs)
Window Functions (RANK(), ROW_NUMBER(), LAG(), LEAD())
Set Operations (UNION, INTERSECT, UNION ALL)
Aggregation & Grouping (SUM, AVG, GROUP BY, HAVING)
📈 Key Analysis Performed
Identified top 5 customers by revenue
Analysed best-performing products
Detected inactive customers
Calculated average order value (£477)
Identified high-value orders (>£1000)
Analysed monthly sales trends
💡 Key Insights
A small number of customers generate a large portion of revenue
Electronics products (e.g., Smart TV, MacBook Air) perform best
Some products show low sales and may need improvement
Sales vary across months, with peak performance in February
Opportunities exist to re-engage inactive customers
